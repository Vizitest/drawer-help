Creating Equivalence Classes

An Equivalence class is a set of data, whose Representative Values are all considered to yield the same result when used for a method input parameter. When used in the Output section of the component, it contains a set of values, all of which are considered valid.

Adding Representative Values

Representative Values are children of Equivalence Classes. Often, a single value within an Equivalence Class is sufficient for Test Case creation. Values will be available in the Test Case Table.

Return Values

The Return Values deal with the data returned by the method. These are also structured as Equivalence Classes and Representative values.

Exceptions

If the method throws an Exception, you can choose from a list of all Exceptions or use the general Exception for any exception.
