The Instance Component instantiates a class that is used in your Test Configuration. It uses Equivalence Class Partitioning, a powerful and logical way of defining test case data.

Quick Overview

The Instance Component is automatically added if you add a Method Component that references a non-static method. If your method does not need to reference the Instance Data, then you do not need to configure the Equivalence Classes in the Instance Component.

If you do need to instantiate the a Class, then you should configure Equivalence Classes and Representative values. Understanding this concept is important to using Vizitest in general. You should take the time to understand Equivalence Class Partitioning. Once understood it will quickly become second nature.

Equivalence Class Partitioning

A technique for dividing input and output data into sets of equivalent values. This is useful for generating test cases. This will make a lot more sense if you watch the video.

Working with Equivalence Classes

In an Instance Component, you would typically have a valid and an invalid Equivalence Class but you can add more if required. You can then add Representative Values to these.

Working with Representative Values

Representative Values are children of Equivalence Classes. Often, a single value within an Equivalence Class is sufficient for Test Case creation. In this component, values are used for Class instantiation and will be available in the Test Case Table.
