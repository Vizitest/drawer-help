An Instance Component is inserted either by directly searching for and inserting a class or, more usuually, by inserting a non-static method.

Instance Component - Docs

Equivalence Class Partitioning

Vizitest uses this powerful technique extensively. The docs link contains more information about Equivalence Class Partitioning.

Adding Equivalence Classes

In an Instance Component, you would typically have at least a valid and an invalid Equivalence Class but you can add more if required. Here we add several.

Adding Representative Values

Representative Values are children of Equivalence Classes. Often, a single value within an Equivalence Class is sufficient for Test Case creation. In this component, values are used for Class instantiation and will be available in the Test Case Table.

Adding an individual Test Case

In many cases, you will want to test with specific cases you choose rather than asking Vizitest to generate them for you. Click the docs link for more information.

Automatically generating Test Cases

Vizitest uses the best-practice "Pairwise Combination" algorithm to automatically generate Test Cases for you. It generates the optimal number of test cases, not every possible test case. The more Representative Values you have, the more Test Cases you will end up with.

Generating and running test code

Once you've matched your inputs and outputs, you're ready to generate and run your test code.
