The Test Case table consists of columns, each of which contains a Test Case. Test Cases can be added manually or automatically.

The data (inputs and outputs) used for a Test Case is determined by the Equivalence Classes and Representative Values you configured in your Instance and Method components.

Overview

A technique for dividing input and output data into sets of equivalent values. This is useful for generating test cases. This will make a lot more sense if you watch the video.

Adding a manual Test Case

In many cases, you will want to test with specific cases you choose. This is not as thorough as the automatic approach, but it is a good way to start, so you see what is involved in selecting inputs and outputs.

Automatically generating Test Cases.

Vizitest uses the best-practice "Pairwise Combination" algorithm to automatically generate Test Cases for you. It generates the optimal number of test cases, not every possible test case. The more Representative Values you have, the more Test Cases you will end up with.

Input Selection

If you add manual test cases, then you should first select the Inputs data you want to use. When you click in an Input test case cell, you will see the available values, as defined in your Instance or Method components.

Output Matching

Both automatically and manually created test cases require that, for each test case, you select an Output value that is expected for the corresponding inputs. You cannot generate your test code until all outputs are matched.

Renaming Test Cases

Test case names are created automatically. There is no need to rename them but the benefit of doing so is clearer labelling in your test execution output.

Generating test code

The final step is to generate your test code. To do this, you press the Generate Code button in the Test Case Table. This will only work once you have got at least one test case, set the necessary inputs and matched the outputs.
