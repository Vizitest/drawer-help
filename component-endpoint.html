User Guide

The above video explains the main concepts. The User Guide walks you through the creation of several endpoints tests. The Test Editor section is a good place to start.

The Header

Core purpose is to set the REST method and the endpoint path. Also used to 'Quick Play' the request and examine the response.

Path variables

Path variables will automatically appear if you include path variables in the endpoint path. For example, users/{uid} will create a uid variable.

Request and Expected Response overview

All sections are optional. Add items to any section by hovering in the section title then pressing the + button

Creating Equivalence Classes

In all sections, the Valid Equivalence Class is automatically created, with a default value, when you add a section item. Hover to get the + Equivalence Class button as shown below.

Adding Values

To add a new value that you want to test, you add it within an equivalence class. Hover on the equivalence class row and press the + Value button. You can then edit the default value that is added to the list.

Scripting

Vizitest lets you write your own pre- and post-request hook scripts in any programming language. You can access and manipulate request and response data and even create your own test cases. Please refer to the documentation for details.
