Overview

The above video explains the main concepts.

Working with APIs

You can add APIs you want to work with, then add servers (development, staging, production etc.) define regularly used headers, security configuration.

OpenAPI

In Feb 2024 it will be possible to import OpenAPI specifications. This will dramatically shorten test configuration times.

Adding a new API

CLick the +API button to define an API. Once defined, you will configure it.

Changing the API name

CLick in the API name to edit it.

Configure Servers

You should create one or more servers for an API. This consists of a friendly name and a hostname and base path.

Configure headers and cookies

You can add any headers and cookies that are regularly used along with default values. You can later choose from these when you configure a test.

Configure security

Security is currently not available but coming soon. This will support common security methods such as Basic Authenticaion, oAuth etc.
